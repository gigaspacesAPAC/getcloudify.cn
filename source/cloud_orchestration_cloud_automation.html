介绍 Cloudify 3.0

使用Cloudify 在云上编排真实的应用

使用Cloudify实现至云的流畅过渡，轻松使甚至最复杂的应用在它们的整个使用周期都自动化。编排您的应用所需的整个云基础设施的创建，从计算机资源开始一直到网络及块存储设备。而后，Cloudify会将您的应用部署到云 (OpenStack 、VMWare vSphere，或者甚至是像Softlayer一样的裸金属），监控它们的进度，并在需要时对它们进行测量。

通过Cloudify，您不会被锁定于某一个特定的云提供商或者某一个类别的云中；在您自己的数据中心或者您选择的云上使用您最喜欢的自动化和配置管理工具部署相同的应用。使用 Cloudify 监控、管理和测量您的应用，无论是什么拓扑结构或者技术堆栈。

在云上智能编排应用。

大多数自动化工具都专注于应用管理的安装和配置阶段，同时，在实际中，许多管理工作都是在应用部署完成后进行的。

Cloudify 为了消除编排和监控之间的界限而进行了重新设计，提供一种使用恰当纠正措施自动对经监控事件作出反应的机制。版本3.0 中包含这些基于一个工作流引擎的自定义工作流的构件，以及一个能够启用任何流程以及堆栈自动化的建模语言。版本 3.1 将推出自动触发此类纠正错误的监控和自定义政策以提供自我修复和自我调整能力。

编排新智能


整合您的整个自动化和监控工具链

一种典型的自动化环境不会以配置管理而结束；它通常涉及多样化和各种不同的工具链以管理供应、配置环境、记录和监控、实时分析，以及工作流自动化。

Cloudify云集了应用使用周期的各个不同阶段所使用的各种各样的工具。

  • 配置管理工具，如：Chef、Puppet、Fabric和Docker
  • 基础设施编排工具，如：OpenStack Heat
  • 记录和监控工具，如：logstash和Elasticsearch
  • 实时分析工具，如：Reimann.io

这样做的话，Cloudify能够推进行业最佳实践惯例，使其易于整合和使用最佳工具以管理您的环境。此外，Cloudify 3.0 还配有一个新的插件架构让您能够轻松整合用于监控、配置管理和云基础设施的范围广泛的工具。


支持VMware、CloudStack、SoftLayer ，以及其它云

Cloudify 3.0 拥有针对 Apache CloudStack 和 VMware vSphere的内置插件，（即将推出vCloud 和SoftLayer）。它还配有一个开放式插件架构以支持诸如Amazon AWS、GCE和CloudStack等的其它云，以及诸如Docker等的Linux容器（将全部在接下来的几周内发布的插件）。现在，通过Cloudify 3.0可以将同一应用跨越多个云环境，无需为每个环境创建一个新Cloudify 设置。这对于从他们的已有环境过渡到一个云环境的用户来说非常有用，也使得OpenStack、VMware、Amazon，以及云之间能够进行突发和简单的混合式云部署。

从设计蓝图到产品

使用Cloudify设计蓝图建模和部署您的应用。为甚至最复杂的应用确定您的应用层和它们之间的依存关系。然后坐下来，让Cloudify为您部署编排。

使用Cloudify的简单设计蓝图使您进入云的步骤自动化，将您的基础结构当作代码。制定出运行应用所需的详细信息，包括其中间件服务，以及它们之间的依存关系、安装、应用和服务的二进制文件所在地方、向外和向内扩展，甚至如何监控整个应用使用周期。

持续交付

您的应用从部署到生产都简单且顺畅。Cloudify 能够使您的DevOps 和持续交付过程自动化，在从部署到UTA再到生产的整个交付过程中顺畅、始终如一地带您进入部署。

完全自动化的应用使用周期

在云上管理应用需要大量移动部件，将所有元件组合在一起经常会很容易出错。Cloudify 能够实现应用在从部署到监控、调整、失效备援、持续交付，以及基础结构升级的整个生命周期的自动化。这些流程的自动化能够在云上管理您的应用时使错误的复杂性和误差幅度都最小化。

轻松升级

Cloudify 支持部署后基础结构升级以及通过我们的设计蓝图或者自定义命令微调。这些自定义命令能够通过Cloudify CLI 或者 REST API 手动或者以编程方式唤起设计图中描述的这些自定义命令。例如，一条自定义命令能够定义如何获取您的代码的新版本并将它部署到您的中间件中。



能够跨越数千个节点的多应用

管理和监控应用需要与应用严密整合以启用快速故障检测、调整，以及其它反应性行为。由于这会使编排任务更加重，许多已有的编排工具都受到它们的监控和编排引擎的限制。这对于可能单个部署会跨越数千实例的大数据或者网络服务来说尤其具有挑战性。

Cloudify 3.0 使用像RabbitMQ一样的消息经纪提供对大型应用的管理和监控服务，通过logstash和Elasticsearch对大型数据与其管理的实例的通信以及一个记录和分析引擎进行管理。

新拓扑技术驱动的监控

许多现有的监控工具只是为了在应用部署后对它们进行监控而设计的。在一个应用持续变化（有时候每天会有数百次）的DevOps环境中，保持监控系统与实际应用部署同步是一项具有挑战性的工作。由于监控系统常常未配备映射应用拓扑结构和架构的功能，因此，需要复杂和容易出错的发现模式映射应用部署及其组件的依存关系。

Cloudify 3.0 推出了一种新概念拓扑驱动监控技术，在这种监控中，整个应用管理和追踪系统都以应用拓扑结构为中心，而不是基础结构。这使得它可以在任何给定时刻不仅追踪应用的状态，还通过一个单一视图追踪部署、更新，以及调整流程的状态。信息由于监控系统与编排引擎相整合，这两个系统始终同步并且保持更新，无需依赖外部发现服务。
