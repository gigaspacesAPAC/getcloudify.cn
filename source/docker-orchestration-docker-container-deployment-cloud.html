介绍使用Cloudify 3.0 插件的Docker Orchestration

内建Docker管理和支持

Cloudify拥有一个内建Docker插件和各种匹配类别，可以让用户利用Docker容器作为任何拓扑结构的一部分。这能够为基于Docker的大型系统提供成熟的编排和自动化方式此外，用于部署您的管理环境的预制Docker图片提供创造性交叉Linux-分配支持以获得更加稳健的体验，更加快速轻松地升级您的管理环境，以及一种更快的引导程序。

智能 Docker 编排 - 将所有一切整合在一起

  • Docker 容器的创建了是为了帮助启用快速、可靠的应用组件或者层部署。这通过创建一个拥有独立随时可部署的应用组件的容器来完成，同时具有成功运行它们所需的中间件和应用业务逻辑。根据设计，Docker 的目的是作为应用的一个独立部分，通常为一层，甚或是一层中的一个节点。
  • 然而，应用的结构通常是多层的，这就意味着，您拥有互相依存的多层，这种依存关系具有从网络连接和远程API调用到应用导之间信息交换等任何性质。因此，应用是由一个具有特定配置的不同容器组成的集合。这也是您为什么需要一种将您的应用的所有部分结合在一起的方法的原因。
  • 正因为此，我们需要 Cloudify - 通过 Cloudify的Docker 编排功能，我们将依据依存关系的顺序处理容器创造的时间，以及所有必要配置，从而允许容器互相沟通和传输所要求的运行时间属性。
  • 根据TOSCA，通过 loudify的基于YAML设计图，您将能够描述甚至是最复杂的拓扑结构，包括：基础设施，以及中间件层，以及它们上面的应用层。Cloudify基本上使用这种TOSCA 编排计划，并使用跨越组件图形或者组件规划图的工作流使这些具体化，并向代理发出命令。而后，这些会创建应用组件并将这些全部结合在一起。

基于Docker的Cloudify管理器设置

Premade Docker 图片会替换当前基于程序包的方法以部署一个Cloudify 管理环境。

这将提供：

  • 一种创造性交叉Linux-分配支持，同时将不同用户提供的图片之间的依存关系树相结合。
  • 更加快速轻松地升级您的管理环境。
  • 更快的引导程序。

这极好地使Manager Blueprint 方法和Docker 以及 Fabric 插件紧密配合以提供一种无缝管理环境体验。
